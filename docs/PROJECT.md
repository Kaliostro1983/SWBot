# WhatsApp Bot — вступна документація

Короткий опис проєкту та правила, щоб швидко зорієнтуватися після перерви або в **новому діалозі з AI**: прикріпіть цей файл або вкажіть шлях `docs/PROJECT.md`.

---

## 1. Призначення

Node.js-бот на базі **whatsapp-web.js** (Puppeteer/Chromium): слухає повідомлення в WhatsApp, маршрутизує їх за **потоками** (`data/flows.json`) або за змінними `.env`, за потреби пересилає дані на **FastAPI**, має веб-панель (статичний UI у `public/`, сервер у `index.cjs`).

---

## 2. Принципи архітектури

Проєкт навмисно **не клонує** сторонні застосунки: з них беруться ідеї вимог (маршрути, RER, фільтри), реалізація залишається простою й прозорою.

- **Транспорт:** бот підключається до WhatsApp, читає повідомлення, застосовує **потоки** (`data/flows.json`) або fallback з `.env`, дотримується лімітів відправки.
- **Розум (RER):** аналіз, правила, `actions` — на боці **FastAPI** (або іншого HTTP API); контракт: POST з текстом і метаданими → відповідь з `actions` (наприклад `send_message`).
- **Еволюція:** спочатку стабільна доставка та логування; далі — розширення полів потоку (ключові слова, медіа, час) **за потреби**; важкі речі (карти, координати) — окремим етапом, якщо реально використовуються.
- **Панель:** Express + статичний UI; при зростанні інтерфейсу можна винести фронт (наприклад Vite/React) без зміни контрактів API.

### Етап 1 — інтеграція WhatsApp (критерії завершення)

Етап вважається **завершеним**, коли перевірено:

1. **Панель:** задані `PANEL_USER` / `PANEL_PASSWORD`, вхід через `/login.html`, вихід з панелі («Вийти з панелі») працюють.
2. **Цикл WhatsApp:** «Запустити бота» / «Увійти (QR)» → за потреби QR → стан **ready** → «Вийти з акаунта» без збоїв.
3. **Скидання:** «Скинути сесію» очищає локальні `.wwebjs_*`; наступний запуск дає новий QR і вхід без ручного видалення папок.
4. **Після зупинки:** «Зупинити» → «Запустити бота» знову піднімає клієнт (якщо сесія збережена, QR може не знадобитися).
5. **Спостережуваність:** статус і журнал (SSE) оновлюються; після перезапуску процесу `node index.cjs` інтерфейс показує актуальний стан.

Далі логічний **етап 2** — автоматизації WA→FastAPI та перевірка контракту з бекендом (див. `data/flows.json`, розділ «Автоматизації» у панелі).

## 3. Структура проєкту

Корінь репозиторію (лише те, що варто тримати під контролем версій; `node_modules` — залежності npm):

| Шлях | Призначення |
|------|-------------|
| `index.cjs` | Головний процес: Express, клієнт WhatsApp, логіка маршрутизації повідомлень, health/log |
| `public/` | Статика веб-панелі (`index.html` та ін.) |
| `data/flows.json` | Збережені потоки (джерело, напрямок, цілі); порожній масив `[]` — режим лише з `.env` |
| `docs/FASTAPI_INGEST.md` | Контракт `POST` на RER з точки зору бота + посилання на `BOT_INGEST_API.md` у `radio_63ombr` |
| `logs/` | Логи рантайму (`bot.log`, `health.json` тощо) — згенеровані файли |
| `run_bot.bat` | Windows: `npm install` за потреби, відкриває браузер на порт, `node index.cjs` |
| `reset_session.bat` | Windows: видаляє `.wwebjs_auth` та `.wwebjs_cache` (скидання сесії WhatsApp) |
| `VERSION` | Версія збірки (один рядок, наприклад `0.1`); показується в панелі та на сторінці входу. **Після завершення кожного наступного етапу** розробки додавати **0.1** до поточного значення (0.1 → 0.2 → 0.3 …) і за бажанням узгоджувати `version` у `package.json` |
| `package.json` / `package-lock.json` | Залежності та скрипти (`npm start` → `node index.cjs`) |
| `.gitignore` | Що не потрапляє в Git (сесія, логи, `node_modules`, `.env`) |
| `.env.example` | Шаблон змінних для копіювання в `.env` (без секретів, у репозиторії) |
| `.env` | Локальні секрети та налаштування (**не комітити**) |
| `.wwebjs_auth/` | Сесія автентифікації WhatsApp (**не комітити**) |
| `.wwebjs_cache/` | Кеш бібліотеки (**не комітити**) |

Залежності (основні): `whatsapp-web.js`, `express`, `axios`, `dotenv`, `qrcode`.

**HTTP API (фрагмент):** `GET /api/state`, `GET /api/logs`, `GET /api/events` (SSE), `POST /api/login|start|stop|logout|reset-session`, `GET /api/chats` (потрібен готовий клієнт), `GET/POST/PUT/DELETE /api/flows`.

**Інжест у FastAPI (RER):** узгодження полів бота з бекендом — **`docs/FASTAPI_INGEST.md`**; канонічний опис API — у сусідньому репозиторії **`radio_63ombr/docs/BOT_INGEST_API.md`**.

## 4. Змінні середовища (`.env`)

| Змінна | За замовчуванням | Опис |
|--------|------------------|------|
| `PORT` | `3001` | Порт HTTP-панелі |
| `FASTAPI_URL` | `http://127.0.0.1:8000/api/ingest/whatsapp` | URL для відправки даних |
| `SOURCE_CHAT` | порожньо | Джерело (чат), якщо **немає** записів у `data/flows.json` |
| `TARGET_CHAT` | порожньо | Чат для відповідей з API, якщо потоків немає |
| `SOURCE_FILTER_KEYWORDS` | порожньо | Разом із `SOURCE_FILTER_FREQUENCIES` — фільтри по тексту повідомлення в режимі лише `.env`; якщо обидва порожні — усе пропускається (як `*`) |
| `SOURCE_FILTER_FREQUENCIES` | порожньо | Див. вище; токени збігаються як підрядки в тексті |
| `SEND_PREFIX` | `#go` | Префікс команди відправки |
| `SEND_DELAY_MS` | `2000` | Затримка між відправками (мс) |
| `HEADLESS` | `1` | Headless-режим браузера (`1` / `0`) |
| `PANEL_USER` | порожньо | Логін веб-панелі; разом з `PANEL_PASSWORD` увімкнює обов’язковий вхід |
| `PANEL_PASSWORD` | порожньо | Пароль панелі |

Якщо **обидва** порожні — панель доступна без логіна (у лог пишеться попередження). Точну поведінку дивитись у `index.cjs`.

---

## 5. Запуск

- **npm:** з кореня проєкту — `npm install`, потім `npm start` (або `node index.cjs`).
- **Windows:** `run_bot.bat` (відкриває `http://localhost:3001` і стартує бота).
- Поруч має бути доступний бекенд за `FASTAPI_URL`, якщо це потрібно для вашого сценарію.

## 7. Правила роботи над проєктом

1. **Мінімальні зміни:** правити лише те, що потрібно для задачі; не роздувати рефакторингом без потреби.
2. **Секрети:** не комітити `.env`, не дублювати токени/ключі в документації та чатах публічно.
3. **Артефакти рантайму:** не комітити `logs/`, `.wwebjs_auth/`, `.wwebjs_cache/`, `node_modules/` — виключено через `.gitignore` у корені.
4. **Стиль коду:** узгоджувати з існуючим `index.cjs` (CommonJS, імена, структура).
5. **Документація:** після важливих змін оновлювати `docs/CHANGELOG.md`; за потреби — цей файл (структура, нові змінні, нові скрипти).
6. **Версія:** після **завершення етапу** збільшувати значення у файлі `VERSION` на **0.1** (див. таблицю структури); у `CHANGELOG` фіксувати номер версії для релізу етапу.
7. **Мова інтерфейсу:** UI в `public/` зараз українською — зберігати узгодженість при змінах.

## 9. Швидкий контекст для AI (новий чат)

- **Стек:** Node.js, CommonJS (`index.cjs`), Express, whatsapp-web.js, axios → зовнішній HTTP API.
