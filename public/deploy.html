Розгортання на новому комп'ютері

Покрокова інструкція встановлення WA·Signal Bridge з нуля. Займає 10–20 хвилин.

🟢
Node.js 18 або новіший
Перевірити: node -v
nodejs.org
🐳
Docker Desktop
Потрібен для Signal-контейнерів
docker.com

💡 ОС: Windows 10/11 (x64). Node.js краще встановлювати через nvm-windows, щоб легко оновлювати версію.

2Копіювання файлів

Скопіюйте папку проєкту на новий ПК. Папку node_modules та .wwebjs_auth копіювати не потрібно — вони створяться автоматично.

whatsapp-bot/
  ├── index.cjsобов'язково
  ├── package.jsonобов'язково
  ├── docker-compose.signal.ymlобов'язково
  ├── .envобов'язково
  ├── public/обов'язково
  ├── signal-bridge/обов'язково
  ├── data/міграція ← flows.json, чат-каталог
  ├── node_modules/ ← НЕ копіювати
  └── .wwebjs_auth/ ← НЕ копіювати (WA-сесія)
⚠️ Папку data/ копіюйте лише якщо хочете перенести існуючі автоматизації та чат-каталог. Якщо починаєте з нуля — вона створюється автоматично при першому запуску.

3Встановлення залежностей

Відкрийте термінал у папці проєкту та виконайте:

Термінал (cmd / PowerShell)
# Перейти до папки проєкту
cd D:\Armor\whatsapp-bot

# Встановити npm-залежності
npm install

Це займе 1–3 хвилини. Після завершення в папці з'явиться node_modules/.

✅ Команда не повинна виводити помилок. Попередження (warn) — це нормально.

4Файл конфігурації .env

Створіть файл .env у корені проєкту (або скопіюйте .env.example і заповніть). Якщо переносите з іншого ПК — просто скопіюйте наявний .env.

Мінімальний .env
PORT=3001
HEADLESS=1

# Захист панелі керування
PANEL_USER=user
PANEL_PASSWORD=your_password

# Signal bridge (локальний Docker)
SIGNAL_API_URL=http://localhost:3002
SIGNAL_POLL_MS=5000

# FastAPI (якщо використовується)
FASTAPI_URL=http://127.0.0.1:8000/api/ingest/whatsapp
Змінна За замовч. Опис
PORT 3001 Порт панелі керування
HEADLESS 1 1 — Chrome у фоні, 0 — з вікном (для дебагу)
PANEL_USER — Логін панелі. Якщо порожній — панель без пароля
PANEL_PASSWORD — Пароль панелі
SIGNAL_API_URL http://localhost:3002 Адреса Signal bridge (Docker)
SIGNAL_POLL_MS 5000 Інтервал опитування Signal (мс)
FASTAPI_URL — URL FastAPI-бекенду (якщо є автоматизації → FastAPI)
SEND_PREFIX #go Префікс для дозволу пересилання
SEND_DELAY_MS 2000 Затримка між відправками WA (мс)

5Запуск Docker-контейнерів Signal

Signal-міст працює через два Docker-контейнери: signal-cli-api (Signal-протокол) і signal-bridge (HTTP-обгортка).

⚠️ Перед виконанням команд переконайтесь, що Docker Desktop запущений і показує статус «Running» в треї.

Відкрийте docker-compose.signal.yml і перевірте номер телефону Signal:

docker-compose.signal.yml — рядок для перевірки
SIGNAL_ACCOUNT_NUMBER=+380XXXXXXXXX  # ← номер облікового запису Signal

Потім запустіть контейнери:

Термінал — у папці проєкту
# Перший запуск (або після змін у docker-compose)
docker compose -f docker-compose.signal.yml up -d --build

# Наступні запуски (без rebuild)
docker compose -f docker-compose.signal.yml up -d

# Перевірити що контейнери запущені
docker ps

Очікувані контейнери у виводі docker ps:

signal-cli-api Signal CLI REST API порт 8080 (внутрішній)
signal-bridge HTTP-міст для Node.js порт 3002 → localhost:3002
💡 При першому запуску на новому ПК Signal потрібно буде прив'язати — дивіться крок 8. Дані Signal зберігаються у Docker volume signal_cli_data і залишаються між перезапусками.

6Запуск основного сервісу

Термінал — у папці проєкту
npm start
# або
node index.cjs

У консолі має з'явитися:

Control panel started at http://localhost:3001
WA Bridge 1.5
Signal poll start ...

Відкрийте браузер: http://localhost:3001

✅ Якщо бачите панель керування — сервіс запущено успішно. Далі підключайте WhatsApp і Signal.

Автозапуск при старті Windows — найпростіший спосіб через bat-файл у папці автозапуску:

start-bridge.bat (покласти у проєкт)
@echo off
cd /d D:\Armor\whatsapp-bot
docker compose -f docker-compose.signal.yml up -d
node index.cjs

7Підключення WhatsApp

При першому запуску (або після очищення сесії) WhatsApp потребує сканування QR-коду.

  • 1 Відкрийте панель → вкладка Інтеграція. Карта WhatsApp покаже статус connecting.
  • 2 Натисніть кнопку «Запустити» (або вона запустилась автоматично). Зачекайте 10–30 секунд поки з'явиться QR-код.
  • 3 На телефоні відкрийте WhatsApp → Налаштування → Пов'язані пристрої → Додати пристрій.
  • 4 Відскануйте QR-код з екрана. Статус зміниться на connected.
💡 Сесія зберігається у папці .wwebjs_auth/. Після першого підключення повторне сканування не потрібне — сервіс підключається автоматично.
⚠️ Не видаляйте папку .wwebjs_auth/ — це призведе до необхідності повторного сканування QR. Не копіюйте цю папку між комп'ютерами — сесія прив'язана до машини.

8Підключення Signal

Signal підключається як пов'язаний пристрій (linked device) до існуючого облікового запису Signal. Основний телефон залишається активним.

  • 1 Переконайтесь, що Docker-контейнери signal-cli-api і signal-bridge запущені (крок 5).
  • 2 Відкрийте панель → вкладка Інтеграція. Карта Signal покаже connecting.
  • 3 Натисніть кнопку 🔗 Прив'язати Signal. Введіть назву пристрою (наприклад wa-bridge) та натисніть Отримати QR.
  • 4 На телефоні відкрийте Signal → Налаштування → Пов'язані пристрої → +. Відскануйте QR-код.
  • 5 Зачекайте 5–15 секунд. Статус Signal зміниться на connected.
✅ Дані Signal зберігаються у Docker volume signal_cli_data і не губляться при перезапуску Docker або ПК. Прив'язка одноразова.
🔴 Якщо видалити Docker volume signal_cli_data — прив'язку доведеться робити знову. Перед видаленням переконайтесь, що це свідоме рішення.

iВикористовувані порти

3001 Панель керування (Node.js) http://localhost:3001
3002 Signal Bridge (Docker) HTTP API між Node.js і signal-cli
8080 signal-cli-rest-api (Docker) Внутрішній — напряму не використовується
8000 FastAPI бекенд (опціонально) Зовнішній сервіс, якщо є автоматизації → FastAPI
⚠️ Якщо порт 3001 або 3002 зайнятий іншим процесом — змініть PORT=3001 у .env та ports: "3002:3002" у docker-compose.signal.yml.

!Часті проблеми

🐳 Signal: «connecting» — Docker не запущений

Відкрийте Docker Desktop, зачекайте статус «Running». Потім перезапустіть контейнери:
docker compose -f docker-compose.signal.yml up -d

📱 WhatsApp не підключається / QR не з'являється
  • Chrome встановлений і доступний за стандартним шляхом
  • Антивірус не блокує Puppeteer / Chrome
  • Спробуйте HEADLESS=0 у .env — побачите вікно Chrome
  • Перевірте логи у панелі → вкладка Логи
🔌 Порт зайнятий (EADDRINUSE)

Знайдіть і закрийте процес що займає порт, або змініть PORT у .env.

# Знайти процес на порту 3001
netstat -ano | findstr :3001
taskkill /PID <номер> /F
📦 npm install: помилка ERESOLVE / peer deps

Спробуйте з прапорцем:

npm install --legacy-peer-deps
📋 Де переглядати логи
  • Панель керування → вкладка «Логи» — всі події в реальному часі
  • Термінал — повний stdout Node.js процесу
  • Docker Desktop → контейнер signal-bridge → Logs
  • Файл logs/signal_raw.ndjson — сирі Signal-повідомлення
